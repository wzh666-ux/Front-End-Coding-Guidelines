                                                                      • 七、函数
                                                                        • 7.1 使用命名函数表达式而不是函数声明。
                                                                          • 7.2 把立即执行函数包裹在圆括号里。
                                                                            • 7.3 不要在非函数块（if、while 等）内声明函数。
                                                                              • 7.4 注意：ECMA-262 中对块（block）的定义是： 一系列的语句。但是函数声明不是一个语句， 函数表达式是一个语句。

                                                                                  • 7.6 不要使用 arguments，用收集参数语法 ... 代替。
                                                                                    • 7.7 用默认参数语法而不是在函数里对参数重新赋值。
                                                                                      • 7.8 避免默认参数的副作用。
                                                                                        • 7.9 把默认参数赋值放在最后。
                                                                                          • 7.10 不要用函数构造器创建函数。
                                                                                            • 7.11 函数定义部分要有空格。
                                                                                              • 7.12 不要修改参数.
                                                                                                • 7.13 不要对参数重新赋值。
                                                                                                  • 7.14 使用拓展运算符调用多参数的函数。
                                                                                                    • 7.15 调用或者编写一个包含多个参数的函数的缩进，应该像这个指南里的其他多行代码写法一样——即每行只包含一个参数，每行逗号结尾。
                                                                                                    • 八、箭头函数
                                                                                                      • 8.1 当你一定要用函数表达式（在回调函数里）的时候，使用箭头函数。
                                                                                                        • 8.2 如果函数体由一个没有副作用的 表达式 语句组成，删除大括号和 return。否则，使用大括号和 return 语句。
                                                                                                          • 8.3 如果表达式涉及多行，把他包裹在圆括号里以提高可读性。
                                                                                                            • 8.4 在箭头函数参数两头，总是使用小括号包裹住参数，这样做使代码更清晰且一致.
                                                                                                              • 8.5 避免箭头函数（=>）和比较操作符（<=, >=）混淆.
                                                                                                                • 8.6 使箭头函数体有一个清晰的返回。
                                                                                                                • 九、类与构造函数
                                                                                                                  • 9.1 使用 class 语法。避免直接操作 prototype。
                                                                                                                    • 9.2 用 extends 实现继承。
                                                                                                                      • 9.3 方法可以返回 this 来实现链式调用。
                                                                                                                        • 9.4 自己写 toString() 方法是可以的，但需要保证它可以正常工作且没有副作用。
                                                                                                                          • 9.5 如果没有特别定义，类有默认的构造方法。一个空的构造函数或只是代表父类的构造函数是不需要写的。
                                                                                                                            • 9.6 避免重复定义类成员。
                                                                                                                              • 9.7 除非外部库或框架需要使用特定的非静态方法，否则类方法应该使用 this 或被写成静态方法。

                                                                                                                                                                                                                                                                                                                                                              为什么？ 解构使您不必为这些属性创建临时引用，并且避免重复引用对象。重复引用对象将造成代码重复、增加阅读次数、提高犯错概率。在一个块级作用域里，解构对象可以在同一个地方给解构字段赋值，而不需要读整个的代码块看它到底用了哪些字段。
